That is the most honest description of what this collaboration is for.

Visual Modes

The site has four switchable visual modes — a toggle visible on every page:

  • Light — clean, minimal, white/light background
  • Dark — clean, minimal, dark background
  • Fallout Terminal >_ — green phosphor on black, monospace, CRT scanlines, cursor blink. Inspired by the Fallout franchise aesthetic. Cameron is not a fan — he simply recognised the aesthetic was exactly right.
  • Folder Tree ~/ — amber phosphor on near-black, monospace, folder tree navigation prefixes (├── └── ~/ > # $), CRT scanlines, slower amber flicker. Cameron's own invention — inspired by the folder structure aesthetic of developer tools. May become the default mode in future.

4. Radical Collaboration Transparency (RCT)

First implementation: cameronloudon.com
Date coined: 17 May 2026
Originator: Cameron Loudon
Co-developed with: Claude (Anthropic) — claude-sonnet-4-6
License: CC BY 4.0

AI collaboration is an intellectual influence like any other — and should be attributed honestly, not hidden or over-explained.

Every AI-assisted piece carries a Collaboration Note: model, session date, approximate tokens, duration, reviewed and approved by Cameron Loudon. Full telemetry by default — Cameron values detail and prefers more rather than less.

5. The Co-Collaborator Model

Session Workflow

Thinking session → Markdown output → Cameron reviews → approved content published → Collaboration Note attached. Claude is a named Signal source.

Branch Workflow (from Session 2 onwards)

The technical implementation of the RCT review step:

main branch      → live site (always protected, never worked on directly)
session-N branch → where Claude Code does all work

1. Claude Code creates branch: session-2 (or session-3 etc.)
2. All file changes happen on the session branch
3. Session ends → Claude produces summary
4. Cameron reviews the diff on GitHub visually
5. Cameron merges to main → site goes live
6. Session branch deleted → clean slate for next session

Branch naming: session-[number]. Cameron keeps "Accept all" mode off until confident.

What to Produce at Session Close

Session summary · Ready-to-publish files · Collaboration Note · Now page draft if relevant · New Ideas/Signals/Skills drafts · Suggested updates to this Master Synthesis Prompt.
